# Data and code bundle — Zero-shot time-series foundation models for intraoperative hypotension prediction

Companion reproducibility bundle for the two-cohort benchmark manuscript
(*Can a zero-shot time-series foundation model rival task-trained models for intraoperative
hypotension prediction? A two-cohort benchmark and the role of covariate-awareness*),
Max Haberbusch, Medical University of Vienna.

This bundle contains everything needed to **regenerate every figure and table** in the manuscript
and supplementary material **from precomputed results, with no model training and no
foundation-model inference**. Open `reproduce_paper.ipynb` and run it top to bottom.

Maintained code repository: https://github.com/CellularSyntax/tirex-vitaldb

---

## Quick start

```bash
python -m venv venv && source venv/bin/activate     # Python 3.11+
pip install -r requirements.txt
jupyter notebook reproduce_paper.ipynb              # run all cells, top to bottom
```

Run the notebook from the bundle root (the folder containing `results/`). Figures are written to
`outputs/figs/paper/` and the project root; every table's numbers are printed inline for
verification against the manuscript. Total run time is a few minutes on a laptop; no GPU is needed.

## Data provenance and terms

The source physiological datasets are **not** redistributed here. They are openly available from
their original providers, subject to those providers' data-use terms:

- **VitalDB** (development cohort) — https://vitaldb.net (also on PhysioNet, doi:10.13026/czw8-9p62).
- **MOVER** (external cohort) — https://mover.ics.uci.edu (UCI; doi via the MOVER release).

This bundle redistributes only **derived artifacts** — our models' forecasts, aggregate metrics,
embeddings, and the figures/tables computed from them — under CC BY 4.0.

**Clinical-data note.** Reproducing the exact subject-level train/test split for Fig 2 and the
decision-curve figure requires knowing which surgical cases belong to the same patient. We ship this
as a minimal two-column crosswalk (`results/vitaldb_case_subject_map.csv`: `caseid`, `subjectid`),
which contains **no demographics or clinical values** — only anonymous grouping identifiers from
VitalDB's public case registry. Table 1 (cohort demographics) is provided precomputed in
`results/tables/Table1_cohort.csv`; recomputing it from scratch requires the full VitalDB clinical
table, which users can download directly from VitalDB. If the crosswalk is removed, the scripts fall
back to a case-level split automatically (a documented, very small deviation from the paper's
subject-level numbers).

No model is trained or run in this bundle; the foundation-model weights (TiRex-2, Chronos-Bolt,
TimesFM-2.5, Moirai-1.1-R) are obtained from their original Hugging Face repositories and are cited
in the manuscript.
